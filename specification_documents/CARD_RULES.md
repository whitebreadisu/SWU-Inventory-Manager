# SWU Card Catalog Rules

Rules governing which variant records are expected in the card catalog for each
set. These rules are enforced by the domain tests in
`backend/app/tests/test_card_domain_rules.py`. A test failure means a variant
is missing from the catalog or an unexpected variant was added.

---

|-----|------|------|------------|--------------|----------|----------|----|

| Jump to Lightspeed | JTL | ✓ | ✓ | ✓ | ✓ | Leaders only | ✓ |

**LAW has no standalone Foil slot.** Cards that are foil in LAW exist only as
Foil Hyperspace (F-Hyperspace) or Prestige Foil.

## Rules by Card Type

### Base cards

| Rarity | Expected variants (non-OP, non-Prestige) | Exception |
|--------|------------------------------------------|-----------|
| Common | Standard, Hyperspace | — |
| Rare | Standard, Foil, Hyperspace, F-Hyperspace | LAW: Standard, Hyperspace, F-Hyperspace (no standalone Foil) |

### Leader cards

| Rarity | Expected variants |
|--------|-------------------|
| Common | Standard, Hyperspace, Showcase |
| Rare | Standard, Hyperspace, Showcase |
| Special (S) | Excluded — starter-deck exclusives with set-specific variant rules |

Showcase is always foil. Showcase cards for Leaders are present in all sets.

### All other cards (Unit, Event, Upgrade)

| Set | Expected variants (non-OP, non-Prestige) |
|-----|------------------------------------------|
| SOR, SHD, TWI, JTL, LOF, SEC | Standard, Foil, Hyperspace, F-Hyperspace |
| LAW | Standard, Hyperspace, F-Hyperspace |

**Excluded from this rule:**
- S-rarity (Special/Starter) cards — starter-deck exclusives with limited variant runs
- Token cards (`type = 'Token'`) — generated by card effects, standard-only by design
- Token Upgrades (card numbers beginning with `T`, e.g. `T01 Experience`) — same reason
- Cards with a blank type — TCGPlayer CSV data gap; variant rules unknown

---

## Notes on Prestige and Organized Play (OP) variants

Prestige and OP variants exist in JTL, LOF, SEC, and LAW but are **not tested**
for completeness because not every card receives a Prestige or OP printing.
Their presence in the catalog is valid wherever they exist; their absence is
not considered an error.

## Known Source Data Quirks

| Set | Card | Issue |
|-----|------|-------|
| SEC | Bardottan Ornithopter (#62) + Willrow Hood (#66) | Share printed card numbers 571 (Foil) and 817 (Hyperspace Foil) — manufacturer error. Both coexist in the catalog. |
| SOR | Sneak Attack (#219) | TCGPlayer CSV is missing the Hyperspace Foil row. Record inserted directly into the DB. |
| JTL | Razor Crest (#223) | CSV uses 'Ride For Hire' for variants vs 'Ride for Hire' for the standard. Resolved via case-insensitive name matching. |
| JTL | Rio Durant (#15) | CSV uses 'Wisecrack Wheelman' for Hyperspace/Showcase vs 'Wisecracking Wheelman' for standard. CSV and DB corrected to match. |
| LAW | Chirrut Imwe (#46) | CSV uses 'Chirrut Îmwe' (accented Î) for the standard and 'Chirrut Imwe' (plain I) for variants. Resolved via Unicode accent normalization. |

## Test File Reference

```
backend/app/tests/test_card_domain_rules.py
  TestBaseCardVariants
    test_common_base_cards_have_standard_and_hyperspace
    test_rare_base_cards_have_standard_foil_hyperspace_and_foil_hyperspace
  TestLeaderCardVariants
    test_common_and_rare_leaders_have_standard_hyperspace_and_showcase
  TestNonLeaderNonBaseCardVariants
    test_all_valid_variants_present
```

Run from inside the backend container:

```bash
docker compose exec backend pytest app/tests/test_card_domain_rules.py -v
```
